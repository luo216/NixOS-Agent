# 长时运行智能体

## 问题背景

真实开发任务通常无法在一次短对话内完成，而是需要经历规划、实现、验证、修正和记录等多个阶段。随着任务时间拉长，智能体容易出现上下文膨胀、执行目标漂移、状态遗失和中途恢复困难等问题，因此系统必须具备长时运行支撑能力。

## 设计思路

系统将长时运行视为一种持续推进的任务过程，而不是单轮问答。为此，每个角色都拥有独立的运行线程与上下文空间，任务状态则通过工作区中的稳定文件进行承载，使执行过程能够跨越多个回合持续保持一致性。

## 状态支撑

当前系统主要依赖 `AGENTS.md`、`planned-features.json` 与 `codex-progress.md` 这三类工作区文件支撑长时运行。

- `AGENTS.md` 用于提供稳定规则与系统边界
- `planned-features.json` 用于承载任务拆分与状态推进
- `codex-progress.md` 用于记录执行过程与阶段结果

这些文件使系统在面对长任务时，不必完全依赖对话上下文记忆，而是可以通过外部状态源维持任务连续性。

## 持续推进机制

在长时运行场景中，系统通过角色分工、任务状态记录和交接机制共同推动任务持续前进。`planner` 负责组织任务，`implementer` 负责逐项落实，`worker` 负责长时间执行与验证。这样一来，系统不仅能够持续工作，还能在不同阶段调用更合适的角色承担当前任务。

## 异常恢复

长时运行任务不可避免地会遇到中断、阻塞和验证失败等情况。系统通过状态记录、thread 恢复和交接机制降低异常对整体任务的影响，使任务在中断后仍可继续推进，而不是完全从头开始。
