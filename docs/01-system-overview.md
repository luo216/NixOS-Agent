# 系统总览

## 项目定位

NixOS-Agent 不是通用聊天系统，也不是单纯的 Codex 调用演示，而是一个面向真实开发与配置任务的本地多角色协作系统。系统以 Codex App Server 作为运行基础，以 workspace 作为执行现场，以前端时间线界面作为状态呈现入口，目标是在复杂任务中形成可持续推进、可监督、可观察的智能体协作流程。

## 核心概念

- 多角色协作
- 长时运行智能体
- 智能调试
- 运行状态可视化

这四项共同构成当前系统的主线。其中专业知识库与技能包并不是独立目标，而是服务于多角色协作与任务执行质量的能力支撑。

## 系统边界

系统当前采用本地前后端分离形态运行。后端基于 Express 与 WebSocket 提供服务，前端基于 React 展示运行状态。底层执行依赖 Codex App Server，每个角色对应一个独立的 client thread。系统不追求任意角色的动态生成，也不把浏览器直接暴露为底层运行时入口，而是通过后端完成状态维护、事件归一化与执行编排。

## 整体结构

从整体上看，系统可分为四个层次：

- 前端展示层：负责显示角色状态、feature 进度、时间线与 git 变化
- 后端协调层：负责请求接收、角色调度、状态维护与对外广播
- Codex 运行时层：负责承载不同角色的独立 thread 执行
- 工作区环境层：负责保存任务状态文件、技能包镜像和实际代码仓库内容

## 核心价值

与传统单智能体、单次命令执行的模式相比，本系统更强调复杂任务中的分工协作、连续执行、过程监督与结果可见性，从而更贴近真实软件开发与配置验证场景。
