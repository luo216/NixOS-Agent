# 运行状态可视化

## 设计动机

在多角色协作和长时运行场景下，用户关心的不只是最终结果，还包括任务当前推进到什么阶段、哪个角色正在工作、执行过程中发生了哪些关键事件，以及代码与工作区产生了哪些变化。因此，系统需要将运行状态以可视化方式呈现出来，而不是只保留纯文本对话。

## 可视化对象

当前系统重点展示以下几类运行信息：

- feature 实现进度
- 当前角色状态
- assistant 输出时间线
- plan、tool call 与运行时状态信息
- git status、git diff 与最近 commit

这些信息共同构成了系统对“执行过程”的观察界面。

## 时间线展示

前端并不直接显示底层运行时原始协议，而是展示经过后端整理后的时间线结构。这样做可以让用户更清楚地区分普通文本、计划信息、工具调用、异常提醒与完成状态，从而提高运行过程的可读性。

## 进度与结果展示

系统通过 feature 面板和 git 面板，将任务推进状态与代码变化结果同时呈现。一方面，用户可以看到某个功能是否处于待处理、执行中或已完成状态；另一方面，也可以直接观察代码差异和提交记录，形成从任务目标到实际改动的可追踪链路。

## 可视化价值

运行状态可视化使系统从“看不到过程的智能体执行”转变为“可观察、可追踪、可复核的协作过程”，从而增强了用户对系统执行行为的理解与控制。
